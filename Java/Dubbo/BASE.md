# 核心特性
## 高性能RPC通信协议

### 通信模型
- 同步RR请求
- 消费端异步请求
- 提供端异步执行
- 消费端请求流
- 提供端响应流
- 双向流式通信

## 自动服务发现

### 服务发现方式
- DUBBO提供的Clinet-Based机制
- 独立的`注册中心组件`，如Nacos、Zookeeper、Consul、Etcd
- 将服务的组织与注册将给`底层容器平台`，如K8s

## 运行态流量管控
> - Dubbo提供了包括负载均衡、流量路由、请求超时、流量降级、重试等策略
> - 多种路由方案：金丝雀发布、A/B测试、路由权重、同区域优先
> - 运行态动态生效修改的流控策略
